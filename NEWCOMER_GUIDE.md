# 新人入门指南（my_notebook）

## 1. 这是一个什么项目
这是一个基于 **Flask + SQLite** 的个人知识库网站，功能分为两块：
- **随笔区**：发布、编辑、删除短文，可带图片。
- **笔记区**：管理 Markdown/PDF 笔记，支持分类、在线编辑、下载、全文搜索、DDL 倒计时。

入口文件是 `app.py`，前端模板在 `templates/`，样式与公共脚本在 `static/`。

## 2. 项目目录结构
```text
my_notebook/
├── app.py                 # Flask 应用入口与全部后端路由
├── requirements.txt       # 依赖列表
├── templates/
│   ├── home.html          # 首页
│   ├── essays.html        # 随笔区页面
│   └── notes.html         # 笔记区页面
└── static/
    ├── style.css          # 全局样式变量与公共布局
    └── common.js          # 暗黑模式等公共前端逻辑
```

运行后还会动态生成：
- `blog.db`：SQLite 数据库（存随笔、DDL、搜索索引等）
- `uploads/`：笔记文件目录（按分类子目录组织）
- `static/snippet_images/`：随笔图片目录

## 3. 你需要优先理解的关键点

### 3.1 应用初始化
- `BASE_DIR` 固定路径，避免部署环境下工作目录问题。
- `UPLOAD_FOLDER`、`SNIPPET_IMAGES` 启动时自动创建。
- `init_db()` 启动时执行：自动建表、补齐旧表字段、做一次历史时间修正。

### 3.2 数据模型（SQLite）
- `essays`：随笔主表（内容、图片、作者、创建时间）。
- `ddls`：截止日期事项。
- `notes_fts`：FTS5 全文搜索索引表（path/title/content/category）。
- `meta`：元信息表（如时区修正标记）。

### 3.3 文件与分类机制
- 笔记文件物理存放在 `uploads/<分类>/<文件>`。
- 系统保留分类 `未分类`，会自动保证存在。
- 路径安全依赖 `is_safe_path()`，防止路径遍历和 symlink/junction 绕过。

### 3.4 搜索机制
- 搜索接口是 `/api/search`。
- 实现上采用 **FTS + LIKE 兜底并集**：FTS 优先，LIKE 补充，兼顾中文匹配体验。

### 3.5 时间与鉴权
- 项目强制使用 `CST(+8)` 生成时间戳，避免服务器时区差异。
- 作者名为“寒食季”的随笔，需要 `HANSHIJI_PASSWORD` 环境变量密码校验。

### 3.6 前端组织
- 每个页面模板里有局部样式（页面独有 UI）。
- `static/style.css` 提供全局变量、亮暗色主题、头部导航通用样式。
- `static/common.js` 负责暗黑模式记忆/自动切换逻辑。

## 4. 推荐学习路径（按顺序）
1. **先读 `app.py` 路由分区注释**：
   - 首页 → 随笔区 → 笔记区 → 分类管理 → 文件下载/预览 → DDL → API。
2. **再读 `templates/notes.html`**：
   - 这是最复杂页面，覆盖了文件管理、在线编辑、搜索、DDL 的交互入口。
3. **结合数据库验证理解**：
   - 本地运行后查看 `blog.db` 的表结构和数据变化。
4. **做 3 个最小实操任务**：
   - 上传一篇 `.md`，编辑并搜索关键字；
   - 新建分类并移动/删除文件（验证路径和索引变化）；
   - 发布一条随笔并上传图片，尝试编辑/删除。
5. **最后再改样式**：
   - 先改 `style.css` 变量，再改页面内样式，避免视觉风格不一致。

## 5. 常见坑位提醒
- 任何新文件读写接口都要走 `is_safe_path()`。
- 任何影响笔记文件增删改的逻辑，都要同步 `notes_fts`。
- 若新增“作者受保护”逻辑，前后端都要验证，不要只做前端校验。
- 修改时间展示前，先确认 `CST` 强制时区与旧数据兼容逻辑。

## 6. 开发与调试建议
- 先创建虚拟环境并安装依赖：`pip install -r requirements.txt`。
- 启动：`python app.py`（默认监听 `0.0.0.0:5001`）。
- 用浏览器同时验证：
  - `http://127.0.0.1:5001`（本机）
  - 局域网地址（移动设备）
- 每次改动后至少做一次：
  - 上传/删除笔记
  - 搜索命中验证
  - 随笔发帖与权限验证
